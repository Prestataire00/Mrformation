### 6.3 🟠 P1 — Routes cron e-learning (outline / quiz / chapter)

**Pattern à appliquer aux 3 routes** (`/generate/outline`, `/generate/quiz`, `/generate/chapter`)

**AVANT (faille générique) :**
```typescript
// ❌ courseId direct, pas de validation entity_id

  .from("elearning_courses")

  .single();
```

**APRÈS (fix générique) :**
```typescript
// ✅ Helper de validation entity_id avant toute écriture
async function validateCourseEntityForCron(
  serviceClient: SupabaseClient,
  courseId: string,
  expectedEntityId?: string
) {
  const { data: course, error } = await serviceClient
    .from("elearning_courses")
    .select("id, entity_id, title")
    .eq("id", courseId)
    .maybeSingle();

  if (error || !course) {
    throw new Error(`Course ${courseId} introuvable`);
  }

  if (!course.entity_id) {
    throw new Error(`Course ${courseId} sans entity_id — refusé pour cron`);
  }

  // Si l'appel cron précise l'entity_id attendue (via body/query), vérifier
  if (expectedEntityId && course.entity_id !== expectedEntityId) {
```

### Verdict final

> **Le multi-tenant LMS est à 91,5 % sécurisé côté DB, mais 2 failles P0 critiques dans les routes API admin rendent le système exposé en prod aujourd'hui.**

### Honnêteté du score

- **Ce qui est bon** : le travail de cleanup `allow_all` (6 migrations en mai 2026) a transformé un système initialement à 50 % sécurisé en système à 91,5 %. C'est un excellent travail de fond. Les helpers `public.user_*()` sont bien standardisés.
- **Ce qui reste fragile** : la couche **service_role** reste le maillon faible. La RLS est bypassée par construction sur les routes admin/cron, et **6 routes sur 32 n'ont pas de guard entity_id en code**. C'est une dette de sécurité qui pèse autant que la RLS DB elle-même.

Aucun audit trail ne le détecte aujourd'hui. **C'est l'équivalent fonctionnel d'un IDOR cross-tenant.**

### Recommandations clés

1. **Patcher les 2 P0 sous 48 h** — impact prod immédiat, ~2 h de dev + tests.
2. **Appliquer la migration `audit_rls_fix_2026_06_05.sql`** en staging puis prod — defense-in-depth gratuite.

4. **Tests e2e multi-tenant Playwright** avant la prochaine release majeure (Epic 2.6+).
